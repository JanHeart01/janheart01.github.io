Making websites private and secure.


HTTPS

Hypertext Transfer Protocol Secure (HTTPS) is a network protocol to support secured communications. It is technically the secure version of HTTP, the protocol where data is sent between browsers and websites that a user is connected to. HTTPS means communications between the two are encrypted, protecting any sensitive data. HTTPS uses two security protocols: Transport Layer Security (TLS) or Secure Sockets Layer (SSL).

When compared to Hypertext Transfer Protocol (HTTP), an HTTPS makes sending data easier, and keeps private information safe from being intercepted (i.e. credit card numbers). It provides security for websites by not sharing any personal information. In an HTTPS site, nothing can be intercepted unlike in an HTTP site, leaving eavesdroppers unable to see any of the messages coming between the server and browser.

HTTPS also offers authentication for a website, meaning that it is the site that it says it is rather than an imposter. A "certificate authority" is issued in order to confirm a website's authenticity. Besides authenticity, HTTPS prevents content from being tampered with or blocked by interlopers, which in worst-case scenarios, could potentially result in affecting either the user's computer (hacking), or compromise websites via performing online attacks. In conclusion, a website with HTTPS grants privacy and protection from unwanted hackers.

It may be an ideal network protocol since it is used on most websites, but just like its counterpart, it does have its issues. The obvious one os if it were issued to a third party, whether intentional or not, the website can get hijacked. With TLS/SSL, its load can slow down the Web server's responsiveness. Despite these shortcomings, if used with great care, HTTPS provides safety and privacy for websites and users.

On a separate note, a fun fact I would like to point out are these HTTP error and status codes:


  • 100-199 : informational status
  • 200-299 : success status
  • 300-399 : redirection status
  • 400-499 : client errors
  • 500-599 : server errors
