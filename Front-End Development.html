Front-End Development

Front-End Development and Back-End Development are two jobs associated with web development. Now the front-end refers to the client-side, the part of a website that users interact with. Everything related to this is made of HTML, CSS, and JavaScript, three "languages" that will be discussed in their own individual pages. The back-end is, of course, the server-side, those who are responsible over how the site works, updates, and changes. It basically refers to everything that cannot be seen by the user, such as databases and servers. It seems simple enough, but sometimes the differences can be difficult to notice between the two, but with a little more detail, the differences are clear. Here, front-end development will be the focus.

Front-End Development

Front-end development is the client-side that involves what the user sees. They involve design and coding languages, three languages that front-end developers must know: HTML, CSS, and JavaScript. Is it necessary to know all three of them? I honestly cannot say, but it would be beneficial and might make you more marketable. In fact, those three languages are three of the skills that are in-demand as mentioned here:

Skills To Become A Front-End Developer

Other than markup languages, front-end developers are also familiar with frameworks, the most popular one being Bootstrap. Other frameworks include Foundation, Backbone, and AngularJS in order to create content for any device. These developers are in charge of making sure that the site does not have any errors or glitches that would affect the website's performance.

They are commonly known as "web designers" since they design a website and make sure its design appears as it is supposed to appear on various browsers and devices. The title of web designer is a broad term, however. Front-end developers (or designers) can create a website without assistance from back-end development, but most of them time, they rely on them to build their sites. Front-end developers are the creative types, meaning that they design how the website will appear to the user on the other side, from fonts, colors, drop-down menus, sliders, forms, and so on.

The languages and frameworks that front-end developers use are vast, but I will cover a handful for each category.

Languages


  • JavaScript: With its frameworks and libraries, it’s the core of front-end development, and beyond. It was the first client-side language and is still the most ubiquitous client-side script on the web.
  • HTML5: HTML dictates a site’s organization and content, all interaction aside. HTML elements can annotate footers, headers, how text displays, how media and images appear, and more.
  • CSS3: Broken into modules and comprises the code for every graphic element that make up the look and feel of a website.
  • Also helpful: a CSS pre-compiler like Sass or LESS.
  • AJAX: JavaScript + XML, it allows specific parts of a site to be updated without a full-page refresh by asynchronously connecting to the database and pulling XML-based chunks of data.

Frameworks


  • AngularJS: This framework, and several other JS frameworks, like Backbone.js, leverage the power of JavaScript.
  • jQuery: A fast, small, JS object library that streamlines how JavaScript behaves across different browsers.
  • Bootstrap: Blends HTML, CSS, and JavaScript to facilitate rapid responsive app development, making sites compatible with all modern browsers and on any size screen.
  • Foundation: Used by sites like Facebook, Yahoo!, and eBay.
  • Semantic UI: Focuses on code readability, clean logic and structure, and has tons of features.
  • Pure.css: Created by Yahoo!, this lightweight, small framework is a set of responsive CSS modules to help streamline designing mobile sites.
  • Skeleton.css: Includes the baseline, no-frills foundation for a responsive site, and doesn’t require any compiling.
